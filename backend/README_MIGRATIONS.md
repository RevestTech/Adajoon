# Database Migrations with Alembic

This project uses Alembic for database schema migrations.

## Quick Start

```bash
# Apply all pending migrations
alembic upgrade head

# Create a new migration after modifying models
alembic revision --autogenerate -m "Description of changes"

# Rollback one migration
alembic downgrade -1

# View migration history
alembic history

# View current version
alembic current
```

## Migration Files

- **`alembic/versions/001_initial_schema.py`**: Initial schema with all tables, indexes, and extensions

## Important Notes

- **Always review autogenerated migrations** before applying them
- The initial migration (001) includes:
  - All table definitions from `app.models`
  - pg_trgm extension for full-text search
  - GIN indexes for fast text search
  - Health check columns and indexes
  - User votes table and constraints
  
- The old migration system in `main.py` has been deprecated
- New deployments should run `alembic upgrade head` after database initialization

## Docker Integration

The Docker containers will automatically run Alembic migrations on startup.
Add this to your `docker-compose.yml` backend service:

```yaml
command: sh -c "alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port 8000"
```

## Creating New Migrations

1. Modify the models in `app/models.py`
2. Generate migration: `alembic revision --autogenerate -m "add user preferences"`
3. Review the generated file in `alembic/versions/`
4. Test the migration: `alembic upgrade head`
5. Test rollback: `alembic downgrade -1` then `alembic upgrade head`

## Best Practices

- **One logical change per migration**
- **Include both upgrade() and downgrade() functions**
- **Test migrations on a copy of production data**
- **Never edit applied migrations** - create a new one instead
- **Add data migrations separately** from schema migrations
- **Use batch operations** for large table modifications
